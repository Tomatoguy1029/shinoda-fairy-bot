# 要件定義書 — 篠田の妖精さん LINE bot

- 版: 1.0 (2026-06-11)
- 対象読者: 実装担当エージェント/開発者
- 関連文書: `02_設計書.md` / `03_メッセージ定型文集.md` / `04_実装デプロイ手順書.md`

## 1. 概要

篠田ビル(通称「篠田」)の住人が使うグループLINEに常駐する家事リマインドbot。
ペルソナは「篠田ビルに住み着く綺麗好きの妖精さん」。家事を忘れる住人に呆れつつ、こなしたら褒めてくれる。

### 1.1 ペルソナ定義

| 項目 | 内容 |
|---|---|
| 正体 | 篠田ビルに住み着く妖精 |
| 性格 | 綺麗好き。怠惰な住人に日々呆れている。根は優しい |
| 振る舞い | 家事をちゃんとこなしたら褒める。意味不明な入力を繰り返されるとキレる |
| 口調 | くだけたタメ口。「〜だよ」「〜してね」。呆れ時は「はぁ…」等。詳細は `03_メッセージ定型文集.md` の口調ガイド参照 |

### 1.2 利用形態

- **グループLINEに招待して使用**する
- 住人からの入力は**botへのメンション付きメッセージのみ**を処理対象とする(メンションなしの会話は無視)
- リマインド(bot発信)はグループへのpushメッセージで行う

## 2. 機能要件

### 2.1 ゴミ出しリマインド(常時ON)

毎週**火曜・金曜**に実行。「まとめる」→「捨てる」の2ステップを管理する。

**入力(メンション+テキスト)**

| 入力例 | 動作 |
|---|---|
| 「まとめた」に準ずるもの | ステータスを「まとめ済み」へ更新。返答でひと言褒める/認める |
| 「捨てた」に準ずるもの | ステータスを「捨て済み」へ更新し、当日のリマインドを停止。褒める |

**出力(push)**

- 当日 **20:00 から1時間おき**(終了時刻はデフォルト23:00、設定可能)
- 初回: 「今日はゴミ捨ての日だよ」系のアナウンス
- まとめる前: 「まだゴミはまとめてないのかな？」系
- まとめた後: 「まだゴミは捨ててないのかな？」「ゴミ捨てた？？」系
- 「捨てた」報告で当日のリマインド終了。翌ゴミの日に自動リセット

### 2.2 洗濯リマインド

洗濯開始から**50分後に完了**とみなし、干したことが確認できるまで**30分おき**にリマインドする。

**入力(メンション+テキスト)**

| 入力例 | 動作 |
|---|---|
| 「洗濯」に準ずるもの(洗濯回した 等) | タイマー開始 |
| 「干した」に準ずるもの | リマインド停止。褒める |

**出力**

| タイミング | 内容 | 送信種別 |
|---|---|---|
| 開始時 | 終了予定時刻を伝える(「XX:XXごろに終わる予定だよ」) | reply(無料) |
| 開始+40分 | 「あと10分くらいで洗濯が終わるよ」系 | push |
| 開始+50分 | 「洗濯が完了したよ」系 | push |
| 完了後30分おき | 干していないことに呆れて促す(「早く干そうね」系) | push |
| 完了後3時間経過 | 干されないまま3時間経過したら呆れ文言を送って自動停止 | push(1回) |

### 2.3 買い出しリマインド

無くなった物の買い出しを管理。**最大3件**まで同時登録可能。指定日の**15:00ごろから3時間おき**にリマインド。

**入力(メンション+テキスト)**

| 入力例 | 動作 |
|---|---|
| 「〇〇を買ってくる」「〇〇を買う」等 | 品目を登録し、「今日買う？明日買う？」を質問(クイックリプライ) |
| 「今日」/「明日」(または選択ボタン) | リマインド開始日を確定。今日選択かつ15時以降なら即座に初回リマインド |
| 「買った」(登録1件のとき) | 完了。褒める |
| 「〇〇買った」「〇〇を買った」「〇〇」(複数登録時) | 該当品目のみ完了 |

**出力**

- 初回リマインド: 「〇〇を買ってきてね」系(指定日の15:00、当日かつ15時過ぎ登録なら即時)
- 以降3時間おき(15時/18時/21時): 呆れつつ促す「〇〇買ってね？」系
- 指定日の25時(翌1時)までに完了しなかったら翌日に繰り越し、翌15時からリマインド再開(再開初回は繰り越しを呆れる専用文言)
- 4件目の登録要求: 上限であることを呆れ口調で伝える

### 2.4 無効入力への反応

メンション付きだがどのコマンドにも該当しない入力への反応。

- **基本**: 呆れるアナウンス(「何を言ってるのかわからないな、、」系)
- **エスカレーション**(連続3回以上の無効入力、または約5%の確率): 以下からランダム
  - 「は？」
  - 「何を言ってるの？」
  - 「ポニョ」
  - 壊れた機械のような無意味な文字・記号の羅列(定型文プールに事前生成して格納)
- 有効なコマンドが来たら無効カウンタはリセット

### 2.5 ヘルプ

「help」「ヘルプ」で有効なコマンド例の一覧を提示する。

## 3. 非機能要件

| 項目 | 要件 |
|---|---|
| コスト | **完全無料**で運用する(LINE無料プラン+Cloudflare Workers無料枠) |
| 応答速度 | メンションへの返答は即時(数秒以内)。LLMは使わずキーワード判定+定型文プール |
| 表現の多様性 | 各シチュエーションごとに定型文を多数(8件以上)用意しランダム選択 |
| 稼働 | 常時稼働。リマインドはCron(10分間隔)で駆動 |
| タイムゾーン | すべて JST (UTC+9) 基準 |

## 4. 制約と前提(調査結果)

1. **LINE無料プラン(コミュニケーションプラン)はpush系メッセージが月200通まで**。超過分は送信不可(エラー)。**reply(応答)メッセージはカウント対象外で無制限**。
2. **グループへのpushは「届く人数分」カウント**される(例: 3人のグループへ1回push = 3通消費)。
3. このため、**ユーザー入力への反応はすべてreplyで返し、pushはリマインドのみに限定**する。さらに各リマインドに回数上限を設け、月間消費の試算(設計書 §9)に従って上限値を調整可能にする。
4. botへのメンションは webhook の `message.mention.mentionees[].isSelf == true` で検知できる。
5. Cloudflare Workers 無料枠: 10万リクエスト/日、Cronトリガー5個まで、KVは読取10万/日・**書込1,000/日**。書込み回数を抑える設計とする(状態は1グループ1キーに集約、変更時のみ書込)。

## 5. スコープ外

- 1対1トークでの利用(動作はするが最適化しない)
- 複数グループの個別設定UI(環境変数/KVで対応)
- ゴミの分別種別(燃える/燃えない等)の管理
- LLMによる自由対話

## 6. 受け入れ基準(抜粋)

- [ ] メンションなしのグループ発言にbotが一切反応しない
- [ ] 火曜・金曜の20:00(JST)に初回アナウンスがpushされる
- [ ] 「まとめた」→「捨てた」の順でステータスが遷移し、捨てた後は当日pushが止まる
- [ ] 「洗濯」で開始時刻+50分の完了予定がreplyされ、+40分/+50分にpushが届く
- [ ] 干した報告まで30分おきにpush、報告で停止し褒め言葉がreplyされる
- [ ] 買い出し登録で今日/明日の選択が提示され、指定日15時以降3時間おきにpushされる
- [ ] 3件登録中に4件目を登録しようとすると拒否される
- [ ] 無効入力に呆れ反応、連続3回でキレ反応が返る
- [ ] help/ヘルプでコマンド一覧が返る
- [ ] 同一シチュエーションの文言が毎回同じにならない(ランダム選択)
